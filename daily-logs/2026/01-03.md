## 1ï¸âƒ£ VPC (Virtual Private Cloud)

- A VPC is a **logically isolated private network** in AWS
- You define the **IP address range (CIDR)** (e.g. `10.0.0.0/16`)
- A VPC belongs to **one AWS Region**
- Can span **multiple Availability Zones**
- Used to launch AWS resources like EC2, RDS, Load Balancers

### Security in VPC
- Security Groups (instance-level)
- Network ACLs (subnet-level)

### Important
- Default VPC exists in every region
- Custom VPC provides full control and is preferred for production

- A subnet is a **division of a VPCâ€™s IP range**
- Each subnet belongs to **one Availability Zone**
- Subnets help organize resources by security and availability

### Types of Subnets
**Public Subnet**
- Route to Internet Gateway
- Used for web servers, bastion hosts

**Private Subnet**
- No direct internet access
- Used for databases and backend services

## 4ï¸âƒ£ Security Groups

- Security Groups act as **virtual firewalls**
- Applied at **EC2 / ENI level**
- Control inbound and outbound traffic

### Key Characteristics
- **Stateful**
- Only **ALLOW rules**
- Can reference other security groups

### Best Practice
- Separate security groups for web, app, and database layers
